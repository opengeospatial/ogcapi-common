==== Abstract Test for Requirement `bbox` query parameter definition

[[conf_searchable-collections_bbox-definition,/conf/searchable-collections/bbox-definition]]
[abstract_test]
====
[%metadata]
identifier:: /conf/searchable-collections/bbox-definition
target:: /req/searchable-collections/bbox-definition
test-purpose:: Verify that the Implementation supports the `bbox` query parameter at the `/collections` endpoint, and that it correctly interprets its dimensionality and coordinate reference system.
test-method::
+
--
*Given:* an Implementation conforming to the "Collections" requirements class of OGC API - Common Part 2 - Geospatial Data also claiming conformance to this "Searchable Collections" requirements class +
*When:* performing a GET operation on `/collections` with a valid `bbox` query parameter supplied as a comma-separated array of numbers +
*Then:* +
- assert that the server successfully supports the `bbox` query parameter by returning a `200 OK` response status. +
- assert that the server correctly interprets the bounding box whether it consists of four numbers (2D) or six numbers (3D). +
- assert that when the bounding box consists of four numbers and no `bbox-crs` parameter is provided, the server interprets the coordinates as WGS 84 longitude, latitude (https://opengis.net/def/crs/OGC/1.3/CRS84). +
- assert that when a valid bounding box and a `bbox-crs` parameter are provided, the server interprets the coordinates according to the specified coordinate reference system instead of the https://opengis.net/def/crs/OGC/1.3/CRS84 default.
====

==== Abstract Test for Requirement `bbox` query parameter response filtering

[[conf_searchable-collections_bbox-response,/conf/searchable-collections/bbox-response]]
[abstract_test]
====
[%metadata]
identifier:: /conf/searchable-collections/bbox-response
target:: /req/searchable-collections/bbox-response
test-purpose:: Verify that the server correctly filters the collections in the result set based on spatial intersection with the requested bounding box, while ensuring collections without a spatial extent are preserved.
test-method::
+
--
*Given:* an Implementation conforming to the "Collections" requirements class of OGC API - Common Part 2 - Geospatial Data also claiming conformance to this "Searchable Collections" requirements class +
*When:* performing a GET operation on `/collections` with a valid `bbox` query parameter +
*Then:* +
- assert that the response result set contains all collections whose spatial extent intersects the requested bounding box (considering either the overall bounding box or the more detailed sparse bounding boxes). +
- assert that the response result set excludes any collection whose spatial extent completely falls outside the requested bounding box. +
- assert that the response result set includes any collection that does not describe a spatial extent at all.

==== Abstract Test for Requirement `q` query parameter free-text filtering

[%metadata]
identifier:: /conf/searchable-collections/q
target:: /req/searchable-collections/q
test-purpose:: Verify that the Implementation supports the free-text `q` query parameter, evaluates comma-separated items with a logical OR operator, correctly handles single-term and multi-term match constraints, applies case-insensitive matching, and targets at least the minimum required collection properties.
test-method::
+
--
*Given:* an Implementation conforming to the "Collections" requirements class of OGC API - Common Part 2 - Geospatial Data also claiming conformance to this "Searchable Collections" requirements class +
*When:* performing a GET operation on `/collections` with a valid `q` query parameter +
*Then:* +
- assert that the server successfully processes that request when it includes a `q` parameter consisting of one or more comma-separated strings of characters, returning a `200 OK` response status. +
- assert that the response result set contains only collections whose properties match at least one (logical OR) of the comma-separated items provided in the `q` parameter. +
- assert that an item in the `q` parameter without white space is matched when that exact whole word appears in the collection description properties. +
- assert that an item in the `q` parameter containing multiple terms separated by white space is matched only if those exact terms appear sequentially in the order specified, separated only by white space, within a target collection property. +
- assert that the text matching is performed in a case-insensitive manner. +
- assert that the server evaluates matches against, at a minimum, the `title`, `description`, and `keywords` properties of the collection descriptions.

--
